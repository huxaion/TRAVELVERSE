# Contributing to TravelVerse

Thanks for contributing to TravelVerse.

## Development
1. Fork and create a feature branch.
2. Install dependencies and copy `.env.example` to `.env`.
3. Run `npm run lint` and `npm run build` before opening a PR.

## Guidelines
- Keep changes focused and modular.
- Prefer reusable components.
- Use TypeScript for all new code.
- Avoid hardcoding secrets.
- Use mock data when external APIs are unavailable.

## Pull Request Checklist
- [ ] Feature is documented
- [ ] Lint/build pass locally
- [ ] No secrets committed
- [ ] Added/updated mock data if needed
